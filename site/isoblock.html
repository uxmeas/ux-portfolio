Native Isometric Shapes in Figma. Editable Forever.

14 shape types, isometric math engine, and native Figma vector output. Built so designers never have to flatten an isometric mockup again.

( 01 ) The Problem

Existing isometric plugins output flattened images locked to a preset angle. Designers who need to adjust a shape after creation have to regenerate it from scratch. Every competitor plugin approaches isometric the same way: place a 2D shape into a pre-deformed template frame, then flatten. Edit later? Re-import. Swap the color? Re-export. The workflow treats the isometric shape as a final artifact, not a design element. And creating an isometric shape from scratch in Figma means manual rotation, skewing, and alignment per face. Hours of math for a single data-visualization diagram that any designer could describe in ten words.

( 02 ) How It Was Framed

Competitive analysis surfaced the pattern. Every existing isometric Figma plugin was transform-based: take a 2D shape, apply a pre-deformed template, flatten the output. The opportunity was the plugin no one had built: an isometric math engine that generates native Figma vectors from coordinate geometry, so the shape stays editable after it lands on the canvas.

HMW

How might we give designers isometric shapes inside Figma that stay editable as native vectors, instead of flattened images locked to a preset angle?

( 03 ) The Answer
A / 01

Isometric math engine over preset templates

Coordinate conversion and perspective projection live at the core of the plugin. Every shape is generated by math, not placed into a pre-deformed template. A Box at 100x200x50 dimensions produces the same mathematically accurate isometric projection as one at 40x40x40. The output is always correct. The approach is 553 lines of shape-generation code (`src/shapes.ts`) feeding a 384-line Figma integration layer (`src/code.ts`).

A / 02

Native Figma vectors over flattened output

Every shape inserts as editable Figma vector layers. Designers can modify dimensions, recolor individual faces, or restructure the shape after it lands on the canvas. No re-import, no re-export, no regeneration. This is the core differentiator from every other isometric plugin on the Figma Community.

A / 03

Per-face color controls with live SVG preview

Each face of each shape is independently color-adjustable. The top face of a Box gets one color. The left and right faces each get their own. SVG preview renders the result before insert, so designers see the final output without committing. The UI is a 1640-line React panel (`src/ui.ts`) with shape selector, dimension inputs, color pickers, and a quick-access bar for frequent shapes.

( 04 ) Key Screen Annotations

( 05 ) Results

14 shape types (Box, Cylinder, Sphere, Pyramid, and more) · Native Figma vector output · Isometric math engine with coordinate conversion · Per-face color controls · Live SVG preview per shape · Marketing site live at isoblock.io · Pre-submission to Figma Community

( 06 ) Reflection
What worked

Math-first beat template-first. Every shape is generated from isometric projection math rather than a pre-deformed template, which means the output is always editable and mathematically accurate regardless of dimensions. Native Figma vector output became the real differentiator once the plugin shipped: competitors can replicate feature checklists, but they can't replicate the editable-forever property without rebuilding their entire rendering approach.

What to do differently

Ship the stairs shape first, not last. The stairs implementation went through six fix iterations because the interaction of step count, riser height, and isometric projection got treated as a simpler problem than it actually is. Tackling the hardest shape first would have surfaced edge cases that were re-discovered in every later shape. Also: ship the isometric grid overlay earlier. Designers asked for it from the first preview, and it was held back for later polish. A grid overlay would have anchored early demos in a way the individual shapes alone didn't.
