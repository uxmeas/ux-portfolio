( 01 ) The Problem

5 to 10 minutes per mockup. Creating device mockups for portfolios, presentations, and client work meant leaving Figma entirely. Opening Photoshop, Smartmockups, or another third-party tool, exporting the screen, reimporting the mockup, and placing it back in the layout. Each mockup broke the design workflow twice: once to create the mockup, once to bring it back. The existing Figma plugins address part of this. Mockup Plugin has 1.4 million installs. Clay Mockups has 599,000. Both offer preset angles only. A fixed grid of predetermined views with no ability to adjust perspective freely. Flat mockups with no perspective look amateur. And designers whose compositions don't match any preset angle are stuck.

( 02 ) How It Was Framed

Competitive analysis of the two market leaders identified the gap. Neither offered 4-point free-form perspective editing. Both were preset-based. The opportunity was to build the perspective editor that the market leaders hadn't built. Not just another mockup tool, but the first one with genuine composition freedom inside Figma.

HMW

How might we create perspective-accurate device mockups without leaving Figma, in under 30 seconds, with free-form perspective control instead of preset angles?

( 03 ) The Answer
A / 01

DLT homography over preset-based rendering

Direct Linear Transform (DLT) homography maps screen content onto device frames using mathematically accurate perspective transformation. It's the same math used in camera calibration and augmented reality. Applied here, it enables 4-point perspective dragging. Designers place 4 corner points at any position and the plugin calculates the correct perspective projection. The result is native Figma frames. Not flattened images, not embedded rasters. Everything stays editable after the mockup is created. Competitors achieve their perspective effects differently. By placing screenshots into pre-deformed template frames. Which produces adequate results only when the desired angle matches a preset. DLT renders any angle correctly.

A / 02

Freemium model to validate demand before monetizing

The core 5 device templates (iPhone, MacBook, iPad, Android, Desktop) are free. Pro unlocks additional device types and advanced color configurations at $8. The price point was chosen deliberately. $8 is an impulse purchase for a working designer, low enough that the decision doesn't require justification. Free users build the install base. The install base creates word-of-mouth. Word-of-mouth drives Pro conversions without paid acquisition.

A / 03

Native Figma output over embedded assets

Every mockup generated by MockupKit is a native Figma frame containing the device chrome as vector elements and the screen as a properly masked content frame. Designers can modify device colors, swap screen content, and adjust perspective after creation without regenerating. This is a material difference from mockup tools that output flattened PNG exports. Which require regenerating the entire mockup every time a screen changes.

( 04 ) Key Screen Annotations
SCREEN 01

Perspective Editor

Four corner handles on a transparent canvas. Drag them to any position. The preview updates in real time showing the projected result. No angle presets to browse. No camera views to select. Direct manipulation.

SCREEN 02

Device Controls Panel

Color picker for device body and accent color. Landscape toggle. Quantity stepper for multi-device compositions. All controls are one-click. No nested menus, no dialogs.

( 05 ) Results

30s per mockup · 30KB bundle · 5 device templates · $8 Pro tier · 4-point DLT perspective vs. preset-only competitors · Native Figma frames, fully editable after creation

( 06 ) Reflection
What worked

DLT homography created a genuine differentiator. Perspective freedom is something the 2-million-combined-install competition doesn't offer, which means MockupKit competes on capability rather than discoverability. Freemium validated commercial demand quickly.

What to do differently

More device templates at launch. 5 covers the most common use cases but designers immediately asked for Apple Watch, TV, and custom frame uploads. Requests that suggest a broader template library would have driven faster adoption. Batch generation for portfolio work. Designers building case studies need the same screen in 3 devices at different angles, and generating each mockup individually adds up.
