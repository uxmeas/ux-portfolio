Documentation That Doesn't Break the Session

Voice-first therapy documentation. Speak naturally. AI structures the notes.

Product Designer

( 01 ) The Problem

Typing notes during a session means breaking eye contact. Every therapist faces the same structural tension: documentation is a clinical and legal requirement, but the act of documenting competes directly with the act of being present. The therapist's attention is split between the person in front of them and the screen in their hands. Typing after sessions relies on memory. The 20 minutes between appointments is the only window before the next client arrives. The keyboard is the wrong tool for a clinical context where the most important skill is being present. But the documentation still has to happen.

( 02 ) How It Was Framed

The insight was timing. Therapists don't need to document during sessions or spend evenings on notes. They need to capture observations in the 2 to 3 minute window between sessions. When impressions are fresh, transition time is limited, and the next client is waiting. That window is too short for structured typing. Voice in that window is natural. The design brief became: build for those 2 to 3 minutes.

HMW

How might we help therapists document sessions without disrupting the therapeutic connection, using voice capture with AI structuring observations into clinical records?

( 03 ) The Answer
A / 01

Voice journal removed the keyboard from clinical documentation

The core interaction is one tap to record. Therapists speak observations naturally in the transition window between appointments. "Client presented heightened anxiety this session, mentioned work stress for the third week running, discussed coping strategy review for next session." The AI transcribes, identifies clinical themes, and organizes the content into structured client records without requiring the therapist to format anything. The record is searchable by theme, date, and client. Useful both as session documentation and as longitudinal reference material.

A / 02

Calm design system reduced adoption resistance

Clinical tools are typically designed to look clinical. Dense data, compact layouts, high information density. The premise was that a documentation tool for a wellness context should feel like the wellness context it serves. Soft background colors, generous padding, large touch targets, and minimal cognitive load at every interaction point. The opening screen shows a single record button. The architecture is simple enough that a therapist unfamiliar with AI tools can use it correctly on the first session without instruction.

A / 03

Guided onboarding removed uncertainty before the first recording

Therapists vary significantly in their comfort with AI-mediated documentation. The guided onboarding flow, 4 screens, established what the tool captures, how it's stored, what happens to the voice recording after transcription, and what the AI does with the content. Each screen answered one question. The onboarding didn't train the user on features. It removed the uncertainty that would have prevented a first recording from happening.

( 04 ) Key Screen Annotations
SCREEN 01

Voice Journal Recording Screen

Large central record button. Passive session timer in the upper corner. No text input visible. The screen communicates clearly: the only task here is speaking. Everything else is handled.

SCREEN 02

Client Record with AI-Structured View

Notes organized by session date with theme tags generated from content. Searchable. The AI-generated structure is visible but editable. Therapists can adjust categories and add annotations. The record is theirs. The AI organizes it.

( 05 ) Results

Voice-first documentation flow · Keyboard removed from core interaction · Calm design system · 4-screen guided onboarding · Client records organized by session and AI-detected theme

( 06 ) Reflection
What worked

Voice-first interaction removed the keyboard entirely. The most disruptive element of existing documentation workflows. Without requiring therapists to change how they think about note-taking. Calm design system reduced the tool's perceived clinical burden, making adoption feel lower-stakes.

What to do differently

Validate with licensed therapists before advancing the concept. HIPAA compliance requirements, state licensing board rules around session documentation, and malpractice considerations all shape what AI-mediated documentation can legally do in a clinical context. Those constraints should drive design decisions, not follow them. Prototype the AI structuring accuracy early. The concept's entire value proposition depends on the AI producing clinically useful structure from natural speech. That needs to be tested before UI polish matters.
