## Feature Prioritization Matrix

### RICE Framework
- **Reach**: How many users affected?
- **Impact**: How much value created?
- **Confidence**: How certain are we?
- **Effort**: How much work required?

Score = (Reach × Impact × Confidence) / Effort

### Priority Buckets
1. **Must Have**: Core functionality, blockers
2. **Should Have**: Important features, high value
3. **Could Have**: Nice-to-have, differentiators
4. **Won't Have**: Out of scope, future consideration

## Decision Framework

### Should We Build This Feature?
1. **Customer Value**: Will customers pay for this?
2. **Business Value**: Does it drive our KPIs?
3. **Technical Feasibility**: Can we build it well?
4. **Market Timing**: Is now the right time?
5. **Competitive Advantage**: Does it differentiate us?

Score each 1-5. Build if average > 3.5

## Emergency Protocols

### Major Customer Complaint
1. Acknowledge within 1 hour
2. Investigate root cause
3. Provide resolution timeline
4. Implement fix
5. Follow up personally

### Competitor Major Release
1. Analyze impact
2. Assess our position
3. Communicate to team
4. Adjust roadmap if needed
5. Prepare response strategy
