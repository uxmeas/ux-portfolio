Deep Problem Framing
The Goal

Designers manually recreate document structure in Figma — reading a Word doc or PDF, then building the heading hierarchy, body text, lists, and tables by hand. This takes 30-60 minutes per document with zero creative value.

The Problem

Every designer does this translation work — taking structured content from documents and rebuilding it in a design tool. The content already has hierarchy (H1, H2, body, lists) but designers ignore the structure and recreate it manually because there's no bridge between document format and design format.

How Might We

How might we go from document to design structure in under 60 seconds — with auto hierarchy detection that maps document structure to design tokens?

Before 30-60 min translation Read document, identify hierarchy, manually recreate in Figma. Zero creative value.
After Under 60 seconds Auto-detect structure, map to design tokens, generate Figma layers. Start designing immediately.
User Research
Key Insights
Every designer does this translation

The document-to-design translation is universal — every designer converting content into layouts spends time on structural work before creative work can begin. It's invisible labor that nobody tracks.

Structure matters more than content

Designers don't need perfect content on first pass — they need the hierarchy. Knowing where H1s, H2s, body text, and lists go is 80% of the layout work. Content can be refined later.

Wizard UX reduces overwhelm

A step-by-step wizard that walks through import, detection, mapping, and preview was far less overwhelming than a single-screen power tool. But power users wanted a compact mode too.

Design Approach / Solution

Docs to Design auto-detects document hierarchy — headings, body text, lists, tables — and maps each element to design tokens. The wizard walks designers through import, structure review, style mapping, and preview before generating Figma layers.

For power users, Compact Pro mode condenses the entire flow into a single panel. The brand identity system lets teams save their design token mappings so every document follows the same visual language.

Key Features

1. Structure auto-detection from documents

2. Style mapping to design tokens (headings, body, lists)

3. Live preview before generating layers

4. Compact Pro mode for power users

5. Brand identity system for consistent token mapping

Reflection
✓ What Worked

1. Dual-mode UX — the wizard captured beginners while Compact Pro retained power users who would have bounced from a step-by-step flow

2. Structure-first approach — focusing on hierarchy detection instead of pixel-perfect content rendering meant the tool was useful from day one

◈ What I'd Do Differently

1. Support more input formats earlier — users immediately asked for Google Docs, Notion, and Markdown support beyond Word/PDF

2. Build the SaaS web version in parallel — the Figma plugin validates the concept but the web app reaches non-Figma designers
