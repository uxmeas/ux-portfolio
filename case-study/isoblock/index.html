Deep Problem Framing
The Goal

Isometric art is everywhere — landing pages, app illustrations, marketing materials — but creating it requires advanced 3D skills. Camera angles, lighting, materials, and scene composition all need expertise that most designers don't have.

The Problem

Blender is free and powerful but its learning curve is steep. Designers who want isometric illustrations either spend weeks learning 3D fundamentals or pay for expensive pre-made assets that don't match their brand. There's no middle ground between "learn everything" and "buy pre-built."

How Might We

How might we make isometric art creation accessible to designers without advanced 3D skills — using shape primitives and presets to produce polished scenes in minutes?

Before Complex Advanced 3D skills required. Steep learning curve. Weeks to first usable output.
After Accessible Shape primitives + surface presets = polished isometric scenes in minutes.
Design Approach / Solution

IsoBlock abstracts the complexity of 3D isometric art into a simple panel. Shape primitives (cubes, cylinders, ramps, stairs) snap together like building blocks. Surface presets apply materials with one click. The isometric camera auto-setup handles the math that trips up beginners.

The plugin UI lives in Blender's sidebar panel, following Blender's native UI patterns so it feels like a natural extension rather than a foreign tool bolted on.

Key Features

1. Shape primitives — cubes, cylinders, ramps, stairs, platforms

2. Surface presets — one-click materials (wood, metal, glass, matte)

3. Isometric camera auto-setup — correct angle and orthographic projection

4. Plugin UI panel — native Blender sidebar integration

Reflection
✓ What Worked

1. Abstraction level — hiding 3D complexity behind simple primitives and presets hit the right balance between power and accessibility

2. Native Blender UI patterns — the plugin felt like part of Blender rather than a separate tool, reducing the learning curve

◈ What I'd Do Differently

1. Add a scene template library — starting from a blank canvas is intimidating even with primitives; pre-built starting points would accelerate adoption

2. Export presets for common formats — designers need PNG/SVG output at specific sizes, not Blender render settings
