A Figma plugin that distributes text data across multiple selected text layers. Paste a list from a spreadsheet, each line maps to a layer. Loop, reverse, preview, and undo. Built because I use this workflow every day and wanted it done right.

THE PROBLEM

Populating text
layers is tedious

Every time you design a table, a list, a set of cards, or any repeating UI pattern, you end up copy-pasting text one layer at a time. Select layer, paste, select next layer, paste. 50 names into 50 cards means 100 clicks minimum.

Existing plugins solve this partially but lack preview, undo, and delimiter options. I used Many Paster daily and wanted a version that matched MZM Labs quality standards with the features I was missing.

WHY I BUILT THIS

I built MultiPaste to eliminate the most tedious part of populating realistic designs.

HOW IT WORKS

Three steps

01
Select text layers
Click the layers you want to populate. Nested layers in frames, groups, and components are detected automatically.
02
Paste your data
Drop a list from Excel, a spreadsheet, or type it directly. Choose newline, comma, or tab as delimiter.
03
Preview and paste
See exactly which line maps to which layer before committing. One click to paste, one click to undo.

What I built

Every feature exists because I needed it in my daily workflow. No bloat, no premium tiers.

PLUGIN INTERFACE

Four states

Built from scratch in a single session. Every feature designed, coded, and tested by one person. The plugin I wanted to exist, so I made it.

Async font discovery per text node. Mixed-font nodes require per-character font enumeration before any text write. Stack-based traversal avoids recursion depth limits on deeply nested component instances.

Discriminated union types for all code-to-UI and UI-to-code messages. Type-safe IPC. esbuild bundles TypeScript to ES2017. JS inlined into HTML for Figma's single-file plugin requirement.
